# TeachBooks Documentation Outline

## 1. README.md Structure
- **Title & Abstract:** Formal introduction to the TeachBooks template, highlighting its purpose for Academic and Action-Oriented learning.
- **Problem Statement:** Addressing the challenge of creating interactive, accessible educational materials without extensive web development knowledge.
- **Action-Oriented Setup Guide:** Step-by-step instructions (with visuals) on how to use the template, set up GitHub Pages, and initialize the book.
- **Local Development:** Instructions for building and testing the book locally.
- **Features & Extensions:** Brief overview of capabilities with pointers to the main manual.
- **License & References:** Standard academic reuse and licensing information.

## 2. Book Directory Structure (`book/`)
- **`intro.md`:** Welcome page outlining the pedagogical goals, target audience (University Students, Professional Learners), and a guide on how to navigate the book.
- **`tutorials/` (New directory to replace/reorganize exercises):**
  - **`basics.md`:** Introduction to markdown and Jupyter notebook integration, starting with a common publishing problem and showing the solution.
  - **`interactive_widgets.md`:** A tutorial on using TeachBooks extensions (e.g., syntax exercises, live code) using a Problem-Based Learning approach.
- **`references.md`:** Consolidated bibliography and reference system conforming to academic standards.
- **`credits.md`:** Formal acknowledgments and contributor list.
